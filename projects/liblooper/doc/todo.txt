log增加了vt帮助类
value to text
用于格式化int,float,double
w用法: w=4时,对int是%04d,对float,doubel是%.4f

auto mTag = "fmt";
float v = 0.12345678f;
int i = 123;
logV(mTag) << "i=" << vt(i, 8) << ",float=" << vt(v,2);
//结果为:
//i=00000123,float=0.12
用vt{v,2}也行,比如
logV(mTag) << ",i=" << vt(i, 8) << ",float=" << vt{ v, 2 };

2025.03.27//////////////////////////////////////////////////////////////////////////////////
async/sync分别相当于windows下的PostMessage/SendMessage
zlmediakit作为专用框架，由用户保证不会出现多个EventPoller之间互相嵌套sync
而windows要提供更灵活的通用机制，不同进程/线程中的HWND都必须能向任意HWND SendMessage
所以在SendMessage的处理过程中还要能进行消息循环,而不是像zlmediakit中TaskExecutorInterface::sync简单的sem.wait()
